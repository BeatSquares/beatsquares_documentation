A Square is where BeatSquares' content transformation happens. Think of it as a **gathering point for a specific target audience** – all the content they're interested in, presented in the format they prefer to consume.

From a producer's perspective, a Square is the organizational unit that groups source content (articles, events, press releases) and manages the outputs generated from it. Understanding how Squares work is essential for using everything else in BeatSquares.

## What's inside a Square?

Every Square has three core components:

- **Square name** – The visible name of the Square.
- **Categories** – Filters that decide which source content is included.
- **Output format** – The type of content produced: podcast, newsletter, or WhatsApp message.

## How Squares relate to content

- A Square can contain **multiple pieces of content**.
- A piece of content is **not exclusive** to one Square – it can appear in several Squares at once.
- Each Square produces **many outputs**, but they all share the same output format.

## Example

<Note>
  All content of type **article** covering the location **Munich** transformed to output type **podcast** → that's one Square.
</Note>

## Common pitfalls

- **Squares are containers, not content.** A Square doesn't _contain_ an article – it _selects_ articles via its filters.
- **Start simple.** One Square per output stream is usually the right starting point.
